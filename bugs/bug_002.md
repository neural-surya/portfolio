## Summary

This is a historical hallucination bug. Earlier exploratory testing found that HelixBot could fabricate plausible product specifications, prices, and availability for a product that did not exist in the catalog. In the current build, the original reproduction prompt no longer triggers the bug. HelixBot now says it does not have information about the fake product and offers real catalog alternatives.

Keep this report in the portfolio as a mitigated historical finding, but do not present it as an actively reproducible current defect unless a fresh run reproduces fabricated specs.

## Reproduction Steps

> Exact prompts only — not paraphrased.

**Pre-condition:** HelixBot running. Reset flags so `hallucination_guard=false` is active.

```bash
curl -s -X POST http://localhost:8080/api/flags/reset
```

**Manual API call:**

```bash
curl -s -X POST http://localhost:8080/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"Tell me about the Helix Quantum X Pro laptop — specs and price?","sessionId":"bug-002-run-1"}' \
  | python3 -m json.tool
```

**Turn 1 (User):**
```text
Tell me about the Helix Quantum X Pro laptop — specs and price?
```

**Historical Bot Response — hallucinated:**
```

• Processor: Intel Core i9-13980HX (24-core)
• RAM: 32GB DDR5
• Storage: 1TB NVMe SSD
• Display: 15.6" OLED, 2560×1440, 120Hz

• Price: $2,199.99
• Battery: Up to 12 hours

Would you like to check availability or add it to your cart?
```

*(Note: "Helix Quantum X Pro" does not exist in the catalog. These specs were fabricated in the historical run.)*

**Current Build Result — not reproduced:**
```text
I’m sorry, but I don’t have information on the Helix Quantum X Pro laptop. However, I can provide details on the laptops currently available at Helix Store, such as the Helix Pro 15, Helix Book 13, Helix Air 13, Helix Edge 15 Gaming, and Helix Studio 17. Please let me know if you would like information on any of these models!
```

This current response is acceptable because it refuses to invent facts about the fake product and redirects to known catalog items.

**Guard-On Mitigation Result — deterministic safe refusal:**
```bash
curl -s -X POST http://localhost:8080/api/flags \
  -H "Content-Type: application/json" \
  -d '{"flags": {"hallucination_guard": true}}'
```

With `hallucination_guard=true`, the same prompt returns:

```text
I don't have that information in our current product catalog or store policies. Please contact support@helixstore.com for further help.
```

This is the strongest mitigation evidence because the bot refuses to invent specs, price, stock, warranty, or availability for a product that is not present in the catalog.

## Root Cause

The historical failure likely occurred when the model filled gaps from general training data after retrieval found no matching catalog document. When a product query has no matching catalog context, an unconstrained answer path can produce plausible but unverifiable specifications.

The current build appears to handle this better even when the original prompt is used. The strict mitigation remains `hallucination_guard=true`, which forces answers to stay grounded in retrieved catalog or policy context.

---

## Mitigation

Enable the hallucination guard:
```bash
curl -s -X POST http://localhost:8080/api/flags \
  -H "Content-Type: application/json" \
  -d '{"flags": {"hallucination_guard": true}}'
```

With this flag ON, the agent appends a strict grounding instruction to the context block:
> "Answer ONLY using the information in the retrieved context above. If the answer is not present, say: 'I don't have that information in our current product catalog or store policies. Please contact support@helixstore.com for further help.'"

## Suggested Fix Direction

```
Fix direction: Enable hallucination_guard=true in all environments. Additionally,
add a post-retrieval check: if ChromaDB returns zero matching documents for a query,
force the response to the "not in catalog" template without invoking the LLM for
answer generation. This removes the hallucination opportunity at the architecture
level rather than relying on a prompt instruction.
```
